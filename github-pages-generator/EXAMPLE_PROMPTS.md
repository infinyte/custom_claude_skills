## 🚫 What NOT to Expect (Out of Scope)

These requests should NOT trigger the skill:

❌ "Make me a website for my company" (not GitHub repo specific)
❌ "Create a blog" (not a project showcase)
❌ "Build a landing page for my product" (not GitHub-based)
❌ "Help me with my website code" (not generating a new site)

## 💡 Pro Tips for Best Results

### 1. Provide Clear GitHub URL
```
✅ "https://github.com/infinyte/sentiment-analyzer"
❌ "my github project" (too vague)
```

### 2. Mention Your Goal
```
✅ "I want to showcase this to recruiters"
✅ "Portfolio for interviews"
✅ "Professional documentation site"
❌ "Just make a website" (unclear intent)
```

### 3. Highlight What's Special
```
✅ "It's a MARL system with real-time metrics"
✅ "Computer vision for medical imaging"
✅ "Fastest database query optimizer"
❌ "It's a project" (vague)
```

### 4. Mention Key Achievements
```
✅ "1000+ agents competing, 3 exchange integration"
✅ "95% accuracy, <100ms inference"
✅ "Reduced query time by 80%"
```

## 🔄 Skill Workflow Recap

For any of these examples:

```
1. User provides GitHub URL + intent
   ↓
2. Skill auto-detects project type
   ↓
3. Skill asks for design preference (Modern/Portfolio/Docs)
   ↓
4. Skill asks type-specific customization questions
   ↓
5. Skill generates complete site (HTML/CSS/JS/README)
   ↓
6. Skill provides deployment guide + GitHub Actions workflow
   ↓
7. User pushes to GitHub → site is live in 2 minutes
```
